## 1. Purpose

ControlCore is a local HTTP daemon that acts as a unified gateway for all LLM calls across the workspace. Callers submit structured `ControlCoreCall` payloads specifying intent, target model, and options; the daemon validates via Pydantic, enforces bouncer rules (redaction overrides, determinism seeds, tool safety), routes to the correct adapter (Ollama for local models; OpenAI, Anthropic, xAI, Google, Groq, Together, Mistral, DeepSeek, Perplexity, Cohere for cloud), applies post-model PII/secret redaction, and returns structured results with provenance. A Click CLI provides daemon management and sugar-mode call submission. Used as the LLM backend by `revenue_bot`.

**Data flow:** Client -> `/call` -> Pydantic validation -> Bouncer -> Registry routing -> Adapter execution -> Redaction -> Result with provenance.

**Entry points:** `ControlCore serve` (daemon on 127.0.0.1:8265), `ControlCore run <model> <prompt>`, `python -m ControlCore.daemon`.

**Environment variables:** `CONTROLCORE_HOST`, `CONTROLCORE_PORT`, `OLLAMA_URL`, `CONTROLCORE_REGISTRY`, plus API keys for 10 cloud providers (`OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `XAI_API_KEY`, `GOOGLE_API_KEY`, `MISTRAL_API_KEY`, `GROQ_API_KEY`, `TOGETHER_API_KEY`, `DEEPSEEK_API_KEY`, `PERPLEXITY_API_KEY`, `COHERE_API_KEY`).

## 4. Security Posture

| Item | Status | Notes |
